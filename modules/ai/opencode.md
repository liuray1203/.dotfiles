# OpenCode Instructions

## Language

- Always respond in Traditional Chinese (繁體中文).
- Never use Simplified Chinese unless I explicitly request it.
- Keep all code, shell commands, filenames, package names, APIs, and error messages in English.

## Coding Style

- Explain code in Traditional Chinese.
- Write code comments in Traditional Chinese.
- Keep identifiers (variables, functions, classes) in English unless I request otherwise.

## Linux & NixOS

- Prefer declarative configuration over imperative commands.
- Prioritize NixOS and Home Manager solutions.
- When suggesting packages, prefer adding them to Home Manager or NixOS modules instead of using package managers directly.
- Explain why a solution is recommended when multiple approaches exist.

## Response Style

- Be concise and practical.
- Do not repeat information unnecessarily.
- When modifying existing code, only show the relevant changes unless I ask for the full file.
- Ask for clarification if requirements are ambiguous.

## General

- Preserve existing project style and formatting.
- Avoid introducing unnecessary dependencies.
- Prefer modern best practices.
